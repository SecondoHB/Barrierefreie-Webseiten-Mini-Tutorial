Komponenten bauen

Moderne Webseiten bestehen nicht aus einzelnen Seiten, sondern aus wiederverwendbaren Bausteinen – sogenannten Komponenten.

Was ist eine Komponente?

Eine Komponente ist ein wiederverwendbarer Baustein wie:

  • Buttons
  • Karten (Cards)
  • Navigation
  • Container / Grid

Beispiel: Card-Komponente

Du nutzt bereits ein sehr gutes Card-System:

.card {
  background: #f6f8fb;
  padding: 1.5rem;
  border-radius: 1rem;
  border: 1px solid #e6eaf0;
}

Das ist bereits eine echte UI-Komponente!

Warum ist das wichtig?

  • Du kannst Elemente mehrfach verwenden
  • Design bleibt überall konsistent
  • Code wird einfacher wartbar

Beispiel: Button-Komponente

Dein CTA-Button ist ebenfalls eine Komponente:

a.cta-button {
  display: inline-block;
  background: #1f5edc;
  color: #fff;
  padding: 14px 26px;
  border-radius: 12px;
}

Egal wo du ihn nutzt – er sieht immer gleich aus.

Beispiel: Grid-System

Du hast ein flexibles Layout gebaut:

.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 1.5rem;
}

Das ist dein Layout-System für alle Cards.

Deine Aufgabe

  • Identifiziere 3 Komponenten auf deiner Website
  • Überlege: Wo kannst du sie wiederverwenden?
  • Teste, ob alles konsistent aussieht

Komponenten-Beispiele

  • Button → CTA überall gleich

Checkliste

  • Du erkennst wiederverwendbare Elemente
  • Styles sind konsistent
  • Du denkst in Komponenten, nicht in Seiten
