Animation & UX

Animationen können helfen, Aufmerksamkeit zu lenken – aber nur, wenn sie bewusst eingesetzt werden.

Warum Animationen überhaupt?

  • Sie helfen beim Fokus auf Inhalte
  • Sie machen Übergänge verständlicher
  • Sie verbessern das Nutzergefühl

Das Problem bei Animationen

  • Zu viel Bewegung → verwirrend
  • Flackernde Inhalte → schlecht für Fokus
  • Unkontrollierte Effekte → Barriereproblem

Die richtige Lösung: prefers-reduced-motion

Manche Nutzer möchten keine Animationen sehen (z. B. gesundheitlich): 

@media (prefers-reduced-motion: no-preference) {
  .fade-in {
    animation: fadeInUp 0.8s ease-out;
  }
}

Wie eine Animation funktioniert

@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

Inhalte erscheinen sanft von unten – nicht abrupt.

Wann Animation sinnvoll ist

  • Beim Laden von Inhalten
  • Bei Karten oder Sections
  • Bei kleinen UI-Feedbacks (Hover, Fokus)

Wann Animation schlecht ist

  • Dauerhafte Bewegung im Hintergrund
  • Automatische Slides ohne Kontrolle
  • Wichtige Inhalte, die verschwinden

Checkliste

  • Animationen unterstützen Inhalte
  • Reduced Motion ist berücksichtigt
  • UX bleibt auch ohne Animation gut
